vdom

一、理解 vdom

1、什么是 vdom

  • virtual dom ， 虚拟 DOM
  • 用 JS 模拟 DOM 结构
  • DOM 变化的对比，放在 JS 层来做（图灵完备语言）
  • 提高重绘性能

2、设计一个需求场景

要求：

  • 将该数据展示成一个表格
  • 随便修改一个信息，表格也跟着修改

3、用 jQuery 实现

4、遇到的问题

  • DOM 操作是“昂贵”的，js 运行效率高
  • 尽量减少 DOM 操作，而不是“推倒重来”
  • 项目越复杂，影响就越严重
  • vdom 即可解决这个问题

二、vdom 的应用及核心 API

1、snabbdom

i、h函数
ii、patch函数

2、重做之后的 demo

注：

  • 使用 data 生成 vnode
  • 第一次渲染，将 vnode 渲染到 #container 中
  • 并将 vnode 缓存下来
  • 修改 data 之后，用新 data 生成 newVnode
  • 将 vnode 和 newVnode 对比

3、核心 API

  • h(‘<标签名>’, {…属性…}, […子元素…])
  • h(‘<标签名>’, {…属性…}, ‘….’)
  • patch(container, vnode)
  • patch(vnode, newVnode)

三、diff 算法

1、什么是 diff 算法

  • 是 linux 的基础命令
  • diff命令可以找出两个文本内容的不用的地方。
  • git diff可以查看修改的内容

2、vdom 为何用 diff 算法

  • vdom 中应用 diff 算法是为了找出需要更新的节点
  • DOM 操作是“昂贵”的，因此尽量减少 DOM 操作
  • 找出本次 DOM 必须更新的节点来更新，其他的不更新
  • 这个“找出”的过程，就需要 diff 算法

3、diff 算法的实现过程

  • patch(container, vnode)
  • patch(vnode, newVnode)

  • updateChildren
