前端实践总结

一、HTML

1、HTML特质

高语义、高易用性

2、输入控件水印提示

在IE浏览器(IE9以下)可以使用jquery-placeholder插件

3、表单

  • tabindex
  • placeholder
  • require

4、网页标题

  • 使用标签
  • 只使用一个<h1>标签
  • 标签不能跳级
  • 不要单纯使用标签给内容设置样式

5、H5语义化标签

section、article、aside、hgroup、header、footer、nav、figure、figcaption。

6、H5功能化标签

svg、canvas、audio、video。

7、H5新的输入控件类型

tel、search、url、date、email、number、range、color。

8、base标签

  • <base target="_blank">——默认打开窗口在新窗口打开

9、html5shim框架

<IE9的情况下html5shim框架支持h5新标签

10、Modernizr框架

用于检查是否支持新特性。

二、CSS

1、特点

结构清晰、模块分明

2、命名规则

  • 通用类，分为：
    • default.css(重置样式,ResetCSS和YUI ResetCSS)
    • common.css，分为：
      • 共通模块样式，分为：
        • 对话框、提示框…
        • 页面头部、底部、侧边栏…
      • 基础样式，分为：
        • 页面布局设置
        • 字体设置
        • 背景和前景色
        • 公有class
    • ie-style.css(<　!–[if lt IE 8]>　<　link rel=’stylesheet’ href=’ie-sytle.css’/>　<![endif]–>)
  • 业务类
    • _header.css、_content.css、_footer.css
    • 从上到下，从左到右
    • Less可以帮助你编写出高复用性和高扩展性的CSS代码

3、CSS排序工具

CSScomb

4、基础选择器

  • ID选择器(#reader_title{})
  • 类选择器(.reader_title{})
  • 属性选择器(a[href=www.baidu.com"]{})
  • 伪类和伪对象选择器(:hover{},::after{})
  • 标签类型选择器(a{})
  • 通配符选择器(body *{})

5、选择器组合方式

  • 后代选择符(reader　.title{})
  • 子选择符(.reader>.title{})
  • 相邻选择符(.reader+.title{})

6、选择符权重

ID:(类+属性+伪类):(标签+为对象)

7、一般浏览器默认字体大小

16px

8、高效CSS选择器

  • 避免使用通配符
  • 避免使用标签选择器和单个属性选择器作为关键选择器
  • 不要在ID选择器前使用标签名
  • 尽量不要在选择符中定义过多的层级，最好不要超过三层

9、图片大小

不要超过200kb

10、减少CSS代码量

  • 合并CSS规则(p.title{font:italic bold 12px/30px (microsoft yahei,serif)})
  • 定义简介属性值(#F3E　.8em　0)
  • 合并相同定义

三、JavaScript

1、特点

高可读性、高性能、高安全性

2、提高可维护性

  • 避免定义全局变量或函数，如：
    • 封装在一个变量对象上
    • 通过定义一个匿名函数实现
    • 确保在定义变量时使用var关键字
  • 简化编码方式，在以下情况要注意：
    • 对象创建
    • 数组创建
  • 使用===而不是==
  • 避免使用with语句
  • 避免使用eval
  • 不要编写检测浏览器的代码

3、使用严格的编码方式

  • 不要在全局中使用严格模式
  • 在已有代码中谨慎启用严格模式(JSHint工具可检查JS的编写规范)

4、事件处理和业务逻辑分离

5、配置数据和代码逻辑分离

6、逻辑与结构样式分离

  • 从JS代码逻辑中分离CSS样式
  • 从JS代码逻辑中分离HTML结构，如：
    • 从服务器端动态获取HTML(AJAX)
    • 通过客户端动态生成页面结构(把JS模版代码放在<script>标签中)

7、JS模版

页面中动态的HTMl结构不再由后端生成，后端仅仅提供一套RESTAPI，返回JSON数据。

8、在JS开发中使用MVC模式(AngularJS)或MVVM模式(Vue.js)

9、JS模块化开发

  • CommonJS(Node.js),同步
  • AMD规范(requireJS),异步

10、AJAX

  • 场景，包括：
    • 根据用户需求动态取得后端数据，然后更新界面
    • 不刷新页面而取得任何资源或页面
    • 动态进行用户输入的验证
    • 通过异步方式取得资源
  • 借助AJAX框架，但不要忘记AJAX原生方法
  • 做好和用户交互
  • 使用JSON格式作为AJAX传输的数据格式
  • AJAX请求历史不会被浏览器记录

11、加快JS加载速度

  • 压缩、合并代码
  • 延迟加载
  • 异步加载
  • 动态创建JS

12、良好的编码习惯

  • 嵌套循环时把大循环作为内循环
  • 避免循环内定义变量
  • 在条件分支中创建只在分支中才用到的对象
  • 使用直接量代替对象(使用数字、字符串、布尔值这三个原始类型而不是通过使用Number、Strign、Boolean构造函数创建复杂对象)
  • 缓存计算结果减少重复计算
  • 少用for-in循环
  • 慎用eval
  • 使用数组保存类型单一的数据，其它情况下用对象
  • 正确的内存回收，如：
    • 尽量不使用全局变量
    • 解除不需要的事件监听
    • 不要在闭包中返回外部不需要的对象
  • 尽量不要将外作用域的变量或对象上的属性值缓存在局部变量中，以提高读取性能

13、高效的DOM操作

  • 合并多次的DOM操作作为单次的DOM操作
  • 把DOM元素离线或隐藏后修改，如：
    • 使用文档片段
    • 通过设置DOM元素的display为none来隐藏元素
    • 克隆DOM元素到内存中
  • 设置动画效果的DOM元素的position属性为fixed或absolute
  • 谨慎取得DOM元素的布局信息
  • 使用事件托管方式绑定事件

14、移动设备设置meta标签的viewport属性

<　meta　name=”viewport”　content=”width=device-width,initial-scale=1.0,user-scalable=no”　/>

15、移动设备的多点触摸操作

  • 事件：touchstart、touchmove、touchend、touchcancel(每个事件包含3个列表：touches、targetTouches、changedTouches)
  • active伪类代替hover伪类
  • 虚拟键盘遮挡问题

16、移动设备兼容

  • 使用流式布局
  • CSS　Media　queries(媒体查询技术),如：
    • 智能手机　<480px(@media(max-width:480px){})
    • 竖屏的平板　<768px
    • 横屏的平板　>768px
    • 桌面PC　>1024px
  • 使用合适的图片显示兼容方案，如：
    • img{max-width:100%}
    • 在像素比不同的设备中，加载不同的图片，如：
      • CSS样式中使用Media　queries和image-set
    • 自适应窗口，如：
      • 在html中在img标签中使用srcset属性
      • h5中picture标签
  • 页面简洁，不使用移动设备不兼容的技术，如：
    • 优先使用HTML5　Canvas技术，Flash只是降级方案
    • 避免使用插件(播放、网络连接、存储、访问本地文件等，因为这些在移动设备会失去作用)
  • 设置viewport(<　meta name=”viewport” content=”width=device-width,inital-scale=1”　>)
  • 增加链接和按钮的可操作区域
  • 使用响应式设计框架(Bootstrap框架)
  • 使工具检查移动设备的兼容性(iPadPeek、Howtogomo等)

17、JS判断是否为移动设备

isMobile库和Mobile-Detect库

18、移动前端框架

jQuery　mobile、SenchaTouch

19、字体、行高

字体16px，行高1.5

20、禁用控件选中效果

-webkit-user-select:none;-moz-user-select:none;-ms-user-select:none;user-select:none;

21、在移动设备点击浏览器页面触发的事件

touchstart->touchmove->touchend->mouseover->mousemove->mousedown->mouseup->click

22、标准对话框

alert,confirm,prompt,print,window.open(移动设备避免使用)

23、setInterval()和setTimeout()

setInterval()页面不可见时会停止执行，setTimeout()设定时间不准确。
