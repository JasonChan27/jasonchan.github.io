style-components在项目中遇到的问题

使用style-components在iframe中渲染组件后样式不生效（正式环境中）的问题

一、什么是style-components

styled-components 是一个常用的css in js 类库。和所有同类型的类库一样，通过 js 赋能解决了原生 css 所不具备的能力，比如变量、循环、函数等。

三、排查问题的思路

1、查看style-components生成的style标签，development环境是有text节点的，而production环境是空节点。

如下图：

所以复制style标签到iframe中后，在production环境样式自然是不生效的。

猜测可能原因是打包配置的问题，如下：

  • production环境打包过程中开启了tree-shaking，将插入到style中的逻辑给剔除了。

    • 排除此原因，Tree-shaking主要依赖于ES6的模块化import和export，将没有用到代码去除，而生成的代码中是可以找到相关的代码的。

  • 可能是mini-css-extract-plugin插件将包含 CSS 的 JS 文件提取到了单独的css文件，导致style标签子节点为空。

    • 排除此原因。在其他的css文件中并没找到.test-p相关的样式。

  • 索性将production下的配置改成和development下的配置一样，再次打包，还是同样的效果。

    • 因此，排除了配置对代码逻辑的影响。
2、网上查相关资料，有如下可能
  • 浏览器把插入style的代码当做广告了
    • 排除此原因。取消设置浏览器的广告劫持，还是如此。
  • 安全策略style-src配置的限制
    • 排除此原因。查看请求头的配置，并无此配置。
3、尝试在父document和iframe的document中都渲染Child组件，发现父docuemnt组件样式生效，iframe中的组件样式不生效。并且style标签还是空的。

到此，需要知道style-components的运行机制，所以要去了解下源码。

疑问：为什么development环境下生成的style标签里是有子节点的，而production环境是没有子节点，而是通过CSSOM API来操作样式呢？

尝试将打包配置切换到development环境，并在控制台调试，找到相关的插入样式的代码，关键代码如下:

可知，在development环境下，是通过this.element.insertBefore来添加样式规则的，this.element便是style标签，最后将每条样式规则通过text节点插入到了style标签中。

所以，style-components在development环境下是通过基于文本节点的 CSS 注入系统，以向 DOM 添加样式；而production环境下是通过CSSOM API，以向对应的样式表中添加样式规则。

综上，在development环境下生成的style标签是有对应的样式规则子节点的，然后复制style到iframe中的head中，组件样式可以正常显示；而在production环境下生成的style标签是没有子节点，是通过操作CSSOM AP向样式表中添加样式规则，然后复制style到iframe中的head中，自然组件的样式也不会生效。
