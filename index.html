Node 模块机制

一、Node 简介

1、为什么 Node 的实现语言是 JavaScript

Node.js 创造者 Ryan Dahl，最初是为了设计出具有事件驱动、非阻塞 I/O 特点的高性能 Web 服务器。

JavaScript 作为 Node 实现语言，主要有以下原因：

  • 高性能
    • 当时 Chrome 浏览器的 JavaScript 引擎 V8 性能第一
  • 符合事件驱动
    • JavaScript 在浏览器中有广泛的事件驱动方面的应用
  • 没有历史包袱
    • JavaScript 在后端部分一直没有市场，可以说历史包袱为零，为其导入非阻塞 I/O 库没有额外阻力

React合成事件机制

1、React事件机制是怎样的

React提供了一种顶层注册，事件收集，统一触发的事件机制：

  • 顶层注册 — 在root元素上绑定一个统一的事件处理函数
  • 事件收集 — 在事件触发时，构造合成事件对象，按照冒泡或捕获的路径去组件中收集真正的事件处理函数
  • 统一触发 — 对所收集的事件逐一执行，并共享同一个合成事件对象

深入JavaScript原型链笔记

一、对象属性

ECMAScript中有两种属性：数据属性和访问器属性

1、数据属性

数据属性包含一个数据值的位置。在这个位置可以读取和写入值。

数据属性有4个描述其行为的特性：

  • [[Configurable]]：表示能否通过delete删除属性从而重新定义属性，能否修改属性的特性，或者能否把属性修改为访问器属性。默认值为true。
  • [[Enumerable]]：表示能否通过for-in循环返回属性。默认值为true。
  • [[Writable]]：表示能否修改属性对数据值。默认值为true。
  • [[Value]]：包含这个属性的数据值。读取属性值的时候，从这个位置读；写入属性值的时候，把新值保存在这个位置。默认值为undefined。
