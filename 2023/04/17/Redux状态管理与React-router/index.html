Redux和Vuex状态管理库的实现原理

一、Redux

1、Redux 简介

i、概念

Redux 是一个使用叫做 “action” 的事件来管理和更新应用状态的模式和工具库。它以集中式 Store（centralized store）的方式对整个应用中使用的状态进行集中管理，其规则确保状态只能以可预测的方式更新。

ii、作用和意义

Redux 提供的模式和工具使你更容易理解应用程序中的状态何时、何地、为什么、state 如何被更新，以及当这些更改发生时你的应用程序逻辑将如何表现。 Redux 指导你编写可预测和可测试的代码，这有助于你确信你的应用程序将按预期工作。

iii、使用场景

任何工具都需要权衡利弊，Redux 可帮助你处理共享状态的管理，但会有更多的代码编写，需要添加一些额外代码，并要求你遵循某些限制。

在以下情况下可使用 Redux：

  • 应用中有很多 state 在多个组件中需要使用
  • 应用 state 会随着时间的推移而频繁更新
  • 更新 state 的逻辑很复杂
  • 中型和大型代码量的应用，很多人协同开发

二、Vuex

1、Vuex 简介

i、概念

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式 + 库。借鉴了 Flux、Redux 和 The Elm Architecture。与其他模式不同的是，Vuex 是专门为 Vue.js 设计的状态管理库，以利用 Vue.js 的细粒度数据响应机制来进行高效的状态更新。

ii、作用和意义

通过定义和隔离状态管理中的各种概念并通过强制规则维持视图和状态间的独立性，我们的代码将会变得更结构化且易维护。

iii、使用场景

  • 开发大型单页应用，需要更好地在组件外部管理状态

  • 遍历用户传入的 mutations、actions，基于发布订阅模式实现 store 实例的 mutations 和 actions 方法。
  • 遍历用户传入的 getters，分别包装成一个传入了 store 实例的 state、getters 参数的函数。
  • 将包装后过的 getters，通过Object.defineProperty来重新定义它，并将其定义到 store 实例的 getters 属性上。
  • 通过vue.reactive方法对 state 实现响应式。

3、Vuex3 和 Vuex4 的对比

  • 创建 store 的方法不同。Vuex 3 是通过 new Store() 来创建一个 store 实例，Vuex 4 是通过 createStore 方法来创建。
  • Vuex 4 核心模块导出了 createLogger 函数。
  • 监听数据方式不同。Vuex 3 监听数据是用 new Vue()，而 Vuex 4 使用的是 Vue 3 提供的 reactive 方法。
  • 安装过程不同。Vuex 3 是 Vue.use(Vuex)，Vuex 4 则是 app.use(store)。
  • Vuex4 兼容之前的使用方法。但是在 Vue3 中如果我们想要通过组合式 API 的方法使用 Vuex，我们需要借助与 useStore 这个函数。
