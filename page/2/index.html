Hybrid

一、对 hybrid 的理解

1、解释

  • hybrid 即“混合”，即前端和客户端的混合开发
  • 需前端开发人员和客户端开发人员配合完成
  • 某些环节也可能涉及到 server 端
  • 前端也需了解客户端的知识

2、hybrid 的存在价值

  • 可以快速迭代更新，无需 app 审核(关键)
  • 体验流畅（和 NA 的体验基本类似）
  • 减少开发和沟通成本，双端公用一套代码

异步

一、单线程

只有一个线程，只能做一件事

1、单线程原因：避免 DOM 渲染冲突

  • 浏览器需要渲染 DOM
  • JS 可以修改 DOM 结构
  • JS 执行的时候，浏览器 DOM 渲染会暂停
  • 两段 JS 也不能同时执行（都修改 DOM 就冲突了）
  • webworker 支持多线程，但是不能访问 DOM
